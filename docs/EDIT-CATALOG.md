# Edit the NHANES-style dataset catalog

The homepage is generated by `content/index.jsx`. The editable dataset records are near the top of that file in the `DATASETS` array.

## Add a dataset row

Copy the existing object inside `DATASETS` and update every field:

```js
{
  id: "baseline-questionnaire-release-1",
  release: "Release 1",
  component: "Questionnaire",
  title: "Baseline Questionnaire",
  description: "Participant-level baseline questionnaire responses.",
  documentationHref: "/datasets/baseline-questionnaire-release-1",
  globusPath: "/release-1/questionnaire/baseline/",
  formats: "CSV, SAS7BDAT, PDF",
  access: "Controlled",
  published: "2026-07-23",
  isExample: false,
}
```

Use a unique lowercase `id` containing letters, numbers, and hyphens.

## Add the documentation page

Copy:

`content/datasets/micares-research-data-collection.md`

Rename the copy to match the route in `documentationHref`, for example:

`content/datasets/baseline-questionnaire-release-1.md`

Fill in the release summary, data files, codebook links, variable conventions, access requirements, citation, and contact information.

## Point to the exact Globus folder

Set `globusPath` to the folder that contains only that dataset or release. The homepage JavaScript creates the encoded Globus File Manager link automatically.

Examples:

- `/release-1/demographics/`
- `/release-1/questionnaire/baseline/`
- `/release-2/environmental-measures/`

The path must start and end with `/` and must match the collection exactly.

## Publish a public codebook or PDF

Place the file under:

`content/assets/documentation/`

For example:

`content/assets/documentation/baseline-codebook.pdf`

Link to it from a Markdown page as:

`/documentation/baseline-codebook.pdf`

The generator copies `content/assets/` into the public site during the build.

## Remove the prototype warning

After all displayed records contain approved metadata:

1. Set every `isExample` value to `false`.
2. Remove the warning `<Alert>` block from `content/index.jsx`.
3. Replace the example privacy policy and terms in `static.json`.

## Suggested catalog structure

A practical hierarchy is:

- Release or cycle
  - Component or subject area
    - Dataset or file group
      - Public documentation page
      - Exact Globus folder

This mirrors the useful part of the NHANES experience without copying its visual design or requiring a database.
